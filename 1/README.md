# Инструкции к лабораторным `frontend/1`

## Структура

- `page1.html` — базовая HTML-страница.
- `page2/index.html`, `page2/script.js` — набор задач №2.
- `page3/index.html`, `page3/script.js` — набор задач №3.

## Общие правила в этих лабах

- Ввод делается через `prompt(...)`.
- Вывод делается через `alert(...)` или `console.log(...)`.
- После `prompt` учитывается `null` (нажатие `Cancel`).
- Для чисел используется явное преобразование `Number(...)`.

Примечание: в учебных задачах этого достаточно; для реального интерфейса обычно используют формы и DOM-события.

## Базовые конструкции JavaScript (с нуля)

### Типы данных, которые встречаются в лабах

- `number` — числа, например `12`, `3.14`, `NaN`.
- `string` — строки, например `"Привет"`.
- `boolean` — логический тип: `true` или `false`.
- `null` — специальное значение «пусто».
- `undefined` — значение «не задано».
- `object` — объект с полями, например `{ count: 0 }`.
- `array` — массив (технически тоже объект), например `[1, 2, 3]`.

Примечание: в JS массив — это частный случай объекта, но с особыми методами и поведением.

### `=` и сравнения `==` / `===`

- `=` — присваивание значения переменной.
- `==` — нестрогое сравнение (с преобразованием типов).
- `===` — строгое сравнение (без преобразования типов).

Примеры:
- `5 == "5"` -> `true`.
- `5 === "5"` -> `false`.
- `null === undefined` -> `false`.

Примечание: в учебном и рабочем коде почти всегда используют `===` и `!==`, чтобы избежать скрытых ошибок.

### Что такое `NaN`

- `NaN` означает «не число» (`Not-a-Number`).
- Появляется при некорректном числовом преобразовании, например `Number("abc")`.

Проверка:
- `Number.isNaN(value)` — проверить, что именно `NaN`.
- `Number.isFinite(value)` — проверить, что это нормальное конечное число.

### `null` и почему есть проверка после `prompt`

- `prompt(...)` возвращает:
- строку, если пользователь что-то ввел;
- `null`, если нажал `Cancel`.

Поэтому в коде есть:
- `if (raw === null) return;`

Примечание: без этой проверки код может упасть при вызове строковых методов на `null`.

### `let`, `const`, `function`

- `let` — переменная, которую можно изменить.
- `const` — переменная, которую нельзя переназначить.
- `function` — объявление функции.

Примечание: `const` для массива/объекта не делает их полностью неизменяемыми, меняется только запрет на переназначение ссылки.

### Что такое объект (`object`)

Пример:
- `let Counter = { count: 0, add(v) { this.count += v; } }`

Состав:
- свойства: `count`.
- методы: `add`, `sub`.

Доступ:
- через точку: `Counter.count`.
- через ключ: `Counter["count"]`.

### Что такое `this`

- `this` — ссылка на текущий объект в момент вызова метода.
- В `Counter.add(...)` выражение `this.count` означает поле объекта `Counter`.

Примечание: `this` зависит от способа вызова функции, а не только от места объявления.

### Массивы и методы из лаб

- `push` — добавить элемент в конец массива.
- `pop` — удалить элемент с конца массива.
- `map` — создать новый массив после преобразования каждого элемента.
- `filter` — оставить только элементы, прошедшие условие.
- `sort` — отсортировать массив (меняет исходный массив).
- `join` — склеить массив в строку.
- `split` — разделить строку на массив.

### `sort((a, b) => a - b)` и стрелочная функция

- `(a, b) => a - b` — стрелочная функция-компаратор.
- если результат < 0, `a` идет раньше `b`.
- если результат > 0, `a` идет позже `b`.
- если 0, порядок равный.

Примечание: без компаратора числа сортируются как строки.

### Остаток `%`

- `a % b` — остаток от деления `a` на `b`.

Примечание: в JS для отрицательных чисел остаток может быть отрицательным.

### `...` (rest и spread)

- `function median(...numbers)`:
- `...numbers` — rest-параметр, собирает аргументы в массив.

- `median(...arr)`:
- `...arr` — spread, раскладывает массив в отдельные аргументы.

### `apply`

- `fn.apply(thisArg, arrayArgs)` — вызов функции с массивом аргументов.
- В лабе: `median.apply(null, arr)`.

Примечание: в современном коде чаще используют spread, но `apply` полезно знать.

### Регулярные выражения (regex) в лабах

- `/\s/g`:
- `\s` — пробельный символ.
- `g` — глобально (все вхождения).

- `/\s+/`:
- `+` — один или более пробельных символов подряд.

### Рекурсия

- Рекурсия — функция вызывает саму себя.
- В `deepCopy` рекурсия нужна для обхода вложенных объектов/массивов.

Базовый случай:
- если значение не объект или `null`, вернуть как есть.

## Лаба 2 (`page2/script.js`)

### `task1` — месяц по номеру

#### Инструкция

1. Считать номер месяца.
2. Преобразовать в число.
3. Если число в диапазоне `1..12`, вывести месяц из массива.
4. Иначе вывести сообщение об ошибке.

#### Примечания

- Используется массив месяцев и индекс `month - 1`.
- При нечисловом вводе получится `NaN`, диапазон не пройдет.

### `task2` — первые `n` простых чисел

#### Инструкция

1. Считать `n`.
2. Проверить, что `n` — натуральное число.
3. Реализовать `isPrime(num)` с проверкой делителей до `sqrt(num)`.
4. Перебирать числа от 2, пока не наберется `n` простых.
5. Вывести результат строкой через пробел.

#### Примечания

- Проверка до корня быстрее полного перебора делителей.
- Для больших `n` алгоритм может быть медленным, но для учебных значений подходит.

### `task3` — объект `Counter`

#### Инструкция

1. Создать объект `Counter` с полем `count`.
2. Добавить методы `add(value)` и `sub(value)`.
3. Считать два числа: для `add` и для `sub`.
4. Применить методы и вывести итог `count`.

#### Примечания

- Объект создается внутри функции, поэтому состояние не сохраняется между запусками.
- Проверка `Number.isFinite(...)` отсекает нечисловой ввод.

### `task4` — замена запятых на точки

#### Инструкция

1. Считать строку.
2. Заменить все запятые на точки через `split(",").join(".")`.
3. Вывести результат.

#### Примечания

- Это простой способ глобальной замены одного символа.

### `task5` — проверка на палиндром

#### Инструкция

5. Вывести `Да` или `Нет`.

#### Примечания

- Удаляются только пробелы, не знаки препинания.
- Если нужна проверка фраз с пунктуацией, нужно дополнительное очищение строки.

### `median(...numbers)` и `task3` — вычисление медианы

#### Инструкция

1. Реализовать функцию `median(...numbers)`:
2. Отсортировать числа.
3. Найти середину.

5. Для нечетной вернуть центральный элемент.
6. В `task3` считать массив и вызвать `median`:
7. Через `spread`: `median(...arr)`.
8. Через `apply`: `median.apply(null, arr)`.
9. Вывести оба результата.

#### Примечания

- `spread` — современный способ передачи массива как аргументов.
- `apply` — более старый, но эквивалентный для этой задачи.

### `task4` — проверка скобок

#### Инструкция

1. Считать строку скобок.
2. Создать стек (массив).
3. На `(` делать `push`.
4. На `)`:
5. Если стек пуст, сразу ошибка.
6. Иначе `pop`.
7. После прохода:
8. Если стек пуст — строка правильная.
9. Иначе — неправильная.

#### Примечания

- Это стандартный стековый алгоритм проверки баланса скобок.
- Легко расширяется на разные типы скобок.

### `deepCopy` и `task5` — глубокое копирование

#### Инструкция

1. В `deepCopy(obj)` вернуть примитив/`null` как есть.
2. Если массив — рекурсивно скопировать `map(...)`.
3. Если объект — создать `{}` и пройти `Object.keys(obj)`.
4. Для каждого ключа рекурсивно копировать значение.
5. В `task5` создать объект, скопировать, вывести в консоль.

#### Примечания

- Текущая версия не обрабатывает циклические ссылки.
- Специальные типы (`Date`, `Map`, `Set`) не поддерживаются отдельно.
- Для учебной задачи такой вариант считается базовым и достаточным.

## FAQ по вопросам из разбора

### Что есть кроме `alert` и `prompt`

Встроенные диалоги браузера:
- `alert(message)`
- `prompt(message, defaultValue)`
- `confirm(message)`

Пример `confirm`:
```js
const ok = confirm("Удалить запись?");
if (ok) {
  // подтверждено
} else {
  // отмена
}
```

### Что такое модалки

Модалка (modal dialog) — это окно поверх страницы, которое временно блокирует взаимодействие с основным интерфейсом, пока пользователь не закроет окно или не выполнит действие.

Разница:
- `alert/prompt/confirm` — системные (встроенные) модалки браузера.
- Кастомные модалки — свои окна на HTML/CSS/JS (настраиваемый дизайн, кнопки, поля, анимации).

### `hasOwnProperty` / `Object.hasOwn`

Проверяют, является ли свойство собственным свойством объекта (а не унаследованным через прототип).

Рекомендуемый современный вариант:
```js
Object.hasOwn(obj, key)
```

### Разница между `in` и `hasOwnProperty`

- `key in obj` — ищет свойство и в самом объекте, и в прототипной цепочке.
- `Object.hasOwn(obj, key)` — проверяет только собственные свойства объекта.

### Как проверить только методы

```js
typeof obj[key] === "function"
```

Пример (только собственные методы):
```js
if (Object.hasOwn(obj, key) && typeof obj[key] === "function") {
  // это метод
}
```

### Как проверить только свойства (без методов)

```js
if (Object.hasOwn(obj, key) && typeof obj[key] !== "function") {
  // это не метод
}
```

### Почему в `deepCopy` используется `[]` и `{}`

Нужно сохранять тип структуры:
- если источник — массив, создаем `[]`;
- если источник — объект, создаем `{}`.

Иначе массив превратится в обычный объект с ключами `"0"`, `"1"`, ... и потеряет поведение массива.

### Почему методы не копируются в текущем `deepCopy`

В `deepCopy` есть фильтр:
```js
typeof obj[key] !== "function"
```

Из-за этого копируются только данные (свойства), а функции/методы пропускаются — что и требуется в задаче «без методов».
